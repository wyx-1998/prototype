## 📋 项目概述

本项目是一个安全生产 AI 助手的**高保真交互原型**，展示了基于大语言模型的智能生产力工具在安全生产场景的应用。采用"AI能力+轻量级数据管理"的设计理念,通过对话式交互完成各类安全生产任务。

### 核心价值主张

- **用对话完成工作** - 通过自然语言交互完成各类安全生产任务
- **智能分析辅助** - AI自动识别隐患、分析数据、生成内容
- **轻量化管理** - 提供必要的数据台账,不做复杂业务流程
- **灵活集成** - 支持独立部署或作为能力组件嵌入

---

## 🎯 产品定位

本原型演示了一款智能安全生产助手,核心功能包括:

- **智能对话助手** - 多模态交互,支持法规查询、隐患分析与写作辅助
- **内容创作中心** - 基于文档的沉浸式编辑器,内置 AI 润色与续写工具
- **全维数据台账** - 隐患、检查、题库等多维数据管理与快速检索
- **统计分析看板** - 数据可视化与智能分析

## ✨ 核心功能演示

### 1. 智能对话 (Smart Chat)

**多场景交互能力展示:**

| 场景 | 触发关键词 | 功能展示 |
|------|-----------|---------|
| 法规查询 | `法规`, `查询`, `规定` | 检索《安全生产法》并提供引用卡片 |
| 隐患辨识 | `隐患`, `照片`, `配电箱` | 图像识别分析,生成整改建议 |
| 公文写作 | `写`, `计划`, `大纲` | 生成年度培训计划等公文草稿 |
| 批量巡检 | `批量`, `巡检`, `报告` | 批量分析照片并生成汇总简报 |
| 数据洞察 | `趋势`, `统计`, `分析` | 生成隐患分布统计图表 |
| 资质审核 | `资质`, `审核`, `承包商` | 智能核验证件并生成审核报告 |
| 人员建档 | `人员`, `建档`, `花名册` | OCR识别证件批量生成花名册 |

### 2. 内容创作中心 (Creator Center)

**基于四大作品类型的专业文档创作平台:**

#### Type-1: 培训课件 (Courseware)
- **画布类型**: PPT课件画布
- **应用场景**: 年度安全培训、岗前培训、专项技能培训
- **触发词**: 课件、PPT、演示文稿、教材
- **特色功能**: 
  - 导入知识库
  - 重新生成当前页
  - 生成演讲稿

#### Type-2: 标准公文 (Standard Doc)
- **画布类型**: 长文档画布
- **应用场景**: 安全管理制度、年度工作计划、应急预案
- **触发词**: 计划、制度、规定、预案、大纲
- **特色功能**:
  - 插入法规条款
  - AI 续写建议
  - 导入业务系统

#### Type-3: 业务单据 (Business Form)
- **画布类型**: 报告画布
- **应用场景**: 隐患整改通知书、每日巡检简报、安全月报
- **触发词**: 通知书、整改、月报、报告、简报
- **特色功能**:
  - 查看台账关联
  - 一键分发

#### Type-4: 考试试卷 (Examination)
- **画布类型**: 特殊扩展
- **应用场景**: 员工安全考核、特种作业复审模拟
- **触发词**: 试卷、考试、习题
- **特色功能**:
  - 导入题库
  - 生成解析

**编辑器特性:**
- 浮动大纲导航 - 章节快速定位
- AI 侧边栏辅助 - 实时润色与续写建议
- 上下文感知工具栏 - 根据文档类型动态显示功能按钮
- 所见即所得编辑 - 富文本编辑体验

### 3. 全维数据台账 (Data Ledger)

**多维度数据管理:**

- **隐患台账** - 隐患快照、区域、整改人、状态
- **检查记录** - 检查单号、问题数、完成时间
- **题库管理** - 题型、难度、知识点、通过率
- **培训材料** - 课件、讲义、手册
- **相关方档案** - 承包商、供应商资质管理

**通用功能:**
- 高级筛选与搜索
- 批量导入/导出
- 跨模块联动跳转

## 🚀 快速开始

### 环境要求

- 现代浏览器 (Chrome 90+, Firefox 88+, Safari 14+)
- 无需后端服务,纯前端静态演示

### 运行步骤

1. **克隆仓库**
   ```bash
   git clone <repository-url>
   cd 原型
   ```

2. **启动本地服务器**
   ```bash
   # 方式一: 使用 Python
   cd prototype
   python -m http.server 8000
   
   # 方式二: 使用 Node.js
   npx http-server prototype -p 8000
   
   # 方式三: 使用 VS Code Live Server 插件
   # 右键 index.html -> "Open with Live Server"
   ```

3. **访问应用**
   ```
   http://localhost:8000/index.html
   ```

4. **查看嵌入式演示**
   ```
   http://localhost:8000/embedded_demo.html
   ```

## 🎯 设计理念

### 对话即工作 (Chat as Work)
通过自然语言完成任务,降低学习成本:
- 无需学习复杂菜单和操作流程
- 用说话的方式描述需求
- AI 自动理解意图并执行

### 轻量级台账 (Lightweight Ledger)
数据管理而非流程管理:
- 存储 AI 生成的结构化数据
- 支持查询、统计和导出
- 不做审批、派发等业务流程

### 深度创作模式 (Deep Creation)
从快速生成到精细打磨:
- 对话生成初稿 (快速)
- 创作中心精修 (深度)
- AI 辅助优化 (智能)

---

## ⚠️ 重要说明

### 原型限制

1. **纯前端模拟** - 所有 AI 回复均为预设文本,无真实 API 调用
2. **硬编码数据** - 台账数据、对话历史均为静态 Mock 数据
3. **关键词触发** - 必须使用预设关键词才能触发特定场景
4. **无数据持久化** - 刷新页面后所有操作记录丢失
5. **有限的交互** - 仅实现核心流程,部分按钮为 `alert()` 占位

### 非生产环境

本原型**不可直接用于生产环境**,需要完整开发:
- 后端服务与数据库
- 真实的 LLM API 集成
- 用户认证与权限系统
- 数据持久化与备份
- 安全加固与性能优化
